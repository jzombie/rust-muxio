```
  dev-dep ──> muxio-tokio-rpc-client ──dev-dep──> muxio-tokio-rpc-server
  dev-dep ──> muxio-tokio-ipc-client ──dev-dep──> muxio-tokio-ipc-server
  dev-dep ──> muxio-wasm-rpc-client   ──dev-dep──> muxio-tokio-rpc-server
  dev-dep ──> muxio-rpc-service, muxio-rpc-service-caller, muxio-rpc-service-endpoint
```

Arrow points **only** outward from `muxio-ext-test`. No crate depends on `muxio-ext-test`. Old client crate test files are removed, so no reverse dep exists.

**WASM gating**: `muxio-wasm-rpc-client` deps on `js-sys`/`wasm-bindgen` are non-optional. To avoid issues on non-WASM targets, add `muxio-wasm-rpc-client` behind:
```toml
[target.'cfg(not(target_arch = "wasm32"))'.dev-dependencies]
```
or conditionally skip WASM test modules with `#[cfg(not(target_arch = "wasm32"))]`. IPC and WS have no such constraints.

## Key Design Decisions

1. **Use `muxio-ext-test` as the harness** -- it already exists, has a build.rs pattern, and depends on the server crates
2. **Sub-modules per transport** (not a single generic trait) -- avoids trait complexity, keeps transport-specific server setup readable
3. **Shared helper functions** for server startup, client connection, and assertions -- extract the ~80% common code
4. **Keep transport-specific server setup inline** for state tests -- raw WS/IPC/WASM servers are fundamentally different
5. **WASM bridge stays a helper** (`wasm_helpers.rs`) -- the `RpcWasmClient`'s lack of built-in I/O makes it inherently different at setup time, but identical at call time

## What Stays Transport-Specific
- WASM bridge setup (handle_connect/handle_disconnect, bidirectional byte forwarding)
- Raw server patterns in state tests (raw WS accept, raw IPC listener, WASM bridge)
- `#[cfg_attr]` attributes for non-Unix platforms
- Server event handling (different event types per transport)

## Risks
- WASM deps (`wasm-bindgen`, `js-sys`) increase build time on native targets -- use conditional dev-dependencies
- IPC socket name collisions in concurrent tests -- add PID to names
- Windows CI for IPC/WASM -- preserve existing `#[cfg_attr(windows, ignore)]` annotations
