# Plan: Standardize RPC Type Aliases & Naming Convention

## Goal

Rename and deduplicate existing RPC type aliases across crates, and add an `AGENTS.md` documenting the naming convention.

## Changes

### 1. `core/src/rpc/rpc_internals/rpc_trait.rs` — Add public type aliases

Add three new `pub type` aliases after the existing traits:

```rust
pub type RpcResponseWriter = Box<dyn FnMut(&[u8], bool) + Send>;
pub type RpcResponseBuffer = Arc<std::sync::Mutex<Vec<(Vec<u8>, bool)>>>;
pub type RpcStreamMethodRouter<'a> = Box<dyn FnMut(u64, u32) -> Option<Box<dyn FnMut(RpcStreamEvent) + Send + 'a>> + Send + 'a>;
```

This is the canonical location for all three types (avoids duplication across crates).

### 5. Add `AGENTS.md` at repo root

Document the naming convention:
- All RPC-related structs/traits/enums in `core/src/rpc/` use `Rpc` prefix: `RpcDispatcher`, `RpcSession`, `RpcResponseWriter`, etc.
- Frame-level types in `core/src/frame/` use `Frame` prefix: `FrameCodec`, `FrameKind`, etc.
- Type aliases for complex callback signatures use `Rpc` prefix and live in `rpc_trait.rs`
- All fields on RPC structs use `rpc_` prefix: `rpc_method_id`, `rpc_request_id`, etc.
- Module files use `rpc_` prefix: `rpc_dispatcher.rs`, `rpc_session.rs`, etc.
- Functions use `snake_case`, methods on RPC types use `rpc_` prefix
