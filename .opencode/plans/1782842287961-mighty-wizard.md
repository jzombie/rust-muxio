## Phase 1: Safe Mechanical Refactors

### Issue 3: Replace `.unwrap()` with `.expect()` in production code
**Files:** 
- `extensions/muxio-rpc-service-endpoint/src/endpoint.rs` (4 calls: lines 52, 58, 64, 67)
- `extensions/muxio-rpc-service-endpoint/src/endpoint_interface.rs` (2 calls: lines 187, 209)
- `extensions/muxio-rpc-service-caller/src/caller_interface.rs` (6 calls: lines 112-115, 321)

**What:** Change `lock().unwrap()` → `lock().expect("descriptive msg")`. These are all `std::sync::Mutex::lock()` calls where poisoning is treated as fatal. `expect()` preserves the panic but adds context identifying *which* mutex failed.

### Issue 7: Replace hardcoded hex method IDs in test infrastructure
**Files:**
- `extensions/muxio-ext-test/src/endpoint_helpers.rs` (lines 14, 17)

**What:** Change `ERROR_TEST_METHOD_ID: u64 = 0xBAD` → `rpc_method_id!("__test.error")` and `UNREGISTERED_METHOD_ID: u64 = 0xDEAD_BEEF` → `rpc_method_id!("__test.unregistered.method")`. Verifies no other code references the old hex values.

**File:** `README.md` (after line 370)
