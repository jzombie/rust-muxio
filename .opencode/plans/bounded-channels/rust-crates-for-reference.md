*   **Fila (`faiscadev/fila`)**: An advanced Rust message broker where fair scheduling and per-key throttling are treated as first-class primitives. It demonstrates how to apply fair-queuing mechanics at the application layer to prevent single tenants or aggressive streams from starving a shared system.
*   **HelixRouter (`Mattbusel/HelixRouter-adaptive-async-compute-router`)**: Cited as a primary reference implementation for building adaptive, async DRR schedulers in Rust to achieve strict bandwidth fairness across independent stream queues.
*   **PANDEMONIUM & scx_cake (`wllclngn/PANDEMONIUM`)**: Rust-based Linux CPU schedulers built using the eBPF `sched_ext` framework that utilize DRR to dynamically learn task behaviors and schedule CPU time with absolute fairness.
*   **libp2p (internals)** and **custom Tokio schedulers**: Highlighted in the research objectives as areas to study for implementing fair queuing and traffic shaping algorithms in the Rust asynchronous ecosystem.
